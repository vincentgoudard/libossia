447 /******************************************************************************
448 * *
449 * Transformation functions *
450 * *
451 * TinySpline is a library focusing on transformations. That is, most *
452 * functions are used to transform splines by modifying their state, e.g., *
453 * their number of control points, their degree, and so on. Accordingly, each *
454 * transformation functions specifies an input and output parameter (along *
455 * with the other parameters required to calculate the actual transformation). *
456 * By passing a different pointer to the output parameter, the transformation *
457 * result is calculated and stored without changing the state of the input. *
458 * This is in particular useful when dealing with errors as the original state *
459 * will never be modified. For instance, let's have a look at the following *
460 * code snippet: *
461 * *
462 * tsBSpline in = ... // an arbitrary spline *
463 * tsBSpline out; // result of transformation *
464 * *
465 * // Subdivide 'in' into sequence of bezier curves and store the result *
466 * // in 'out'. Does not change 'in' in any way. *
467 * tsError err = ts_bspline_to_beziers(&in, &out); *
468 * if (err != TS_SUCCESS) { *
469 * // fortunately, 'in' has not been changed *
470 * } *
471 * *
472 * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
473 * changed allowing you to handle the error properly. *
474 * *
475 * Unless stated otherwise, the order of the parameters for transformation *
476 * functions is: *
477 * *
478 * function(input, [additional_input], output, [additional_output]) *
479 * *
480 * 'additional_input' are parameters required to calculate the actual *
481 * transformation. 'additional_output' are parameters to store further *
482 * results. *
483 * *
484 * Note: None of TinySpline's transformation functions frees the memory of the *
485 * output parameter. Thus, when using the same output parameter multiple *
486 * times, make sure to free memory before each call. Otherwise, you will *
487 * have a bad time with memory leaks: *
488 * *
489 * tsBSpline in = ... // an arbitrary spline *
490 * tsBSpline out; // result of transformations *
491 * *
492 * ts_bspline_to_beziers(&in, &out); // first transformation *
493 * ... // some code *
494 * ts_bspline_free(&out); // avoid memory leak. *
495 * ts_bspline_buckle(&in, &out); // next transformation *
496 * *
497 * If you want to modify your input directly without having a separate output *
498 * just pass it as input and output at once: *
499 * *
500 * tsBSpline s = ... // an arbitrary spline *
501 * tsReal *knots = ... // a knot vector *
502 * *
503 * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
504 * *
505 * Note: If a transformation function fails *and* input != output, all fields *
506 * of the output parameter are set to 0/NULL. If input == output, your *
507 * input may have an invalid state in case of errors. *
508 * *
509 ******************************************************************************/
